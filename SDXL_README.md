# Imagene - SDXL on Android

Offline Stable Diffusion XL (SDXL) image generation on Android using ONNX Runtime Mobile.

## Project Structure

```
imagene/
├── app/                              # Android application
│   ├── build.gradle                  # Android build config with ONNX Runtime
│   └── src/main/
│       ├── java/com/micklab/imagene/
│       │   ├── MainActivity.java     # UI and generation workflow
│       │   └── SdxlRunner.java       # ONNX Runtime inference
│       └── assets/models/            # ONNX models (generated by CI)
├── scripts/
│   ├── export_sdxl_onnx.py          # Export SDXL to ONNX
│   ├── optimize_onnx.py             # Optimize ONNX models
│   └── requirements.txt             # Python dependencies
├── .github/workflows/
│   ├── android.yml                  # Basic Android build
│   └── sdxl.yml                     # Full SDXL build pipeline
└── test_sdxl_local.py               # Local testing (UserLAnd)
```

## Requirements

### Target Device
- RedMagic 11 Pro (Snapdragon 8 Elite Gen 5)
- Android 14+
- ~8GB free storage for models

### Build Environment
- GitHub Actions (Ubuntu latest)
- JDK 17
- Gradle 8.7
- Python 3.11

## GitHub Actions Workflow

The `sdxl.yml` workflow performs:

1. **Export ONNX** - Downloads SDXL from HuggingFace and exports to ONNX
2. **Optimize** - Applies ONNX Runtime optimizations
3. **Build APK** - Packages models into Android assets and builds APK

### Manual Trigger

```bash
gh workflow run sdxl.yml
```

### With Options

```bash
gh workflow run sdxl.yml \
  -f skip_export=true \     # Use cached models
  -f quantize=true          # Apply INT8 quantization
```

## Local Development

### Setup Python Environment

```bash
cd scripts
pip install -r requirements.txt
```

### Export Models Locally

```bash
python scripts/export_sdxl_onnx.py \
  --model-id stabilityai/stable-diffusion-xl-base-1.0 \
  --output-dir onnx \
  --device cpu
```

### Optimize Models

```bash
python scripts/optimize_onnx.py \
  --input-dir onnx \
  --output-dir onnx_optimized \
  --mobile
```

### Test on UserLAnd

```bash
pip install onnxruntime Pillow numpy
python test_sdxl_local.py --models-dir onnx_optimized --quick
```

## Android App Features

- Prompt input with negative prompt support
- Adjustable steps (1-50)
- Adjustable guidance scale (0-20)
- 1024x1024 output resolution
- Progress tracking during generation
- Dark theme UI

## ONNX Runtime Configuration

The app uses ONNX Runtime Mobile with:

- **NNAPI** acceleration (Snapdragon 8 Elite NPU)
- **FP16** UNet for reduced memory
- Optimized graph execution

## Performance Notes

Expected generation times on Snapdragon 8 Elite:

| Steps | Time (estimate) |
|-------|-----------------|
| 10    | ~30s            |
| 20    | ~60s            |
| 30    | ~90s            |

## Known Limitations

1. **Model Size** - Total ~7GB, requires large APK
2. **Memory** - Needs ~8GB RAM during inference
3. **First Load** - Models must be copied from assets on first run
4. **Tokenizer** - Simplified tokenizer (BPE would improve quality)
